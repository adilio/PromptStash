# PromptStash improvement plan

## How to use this file

Work through tasks in order. After completing each task:
- Mark it `[x]`
- Add a brief note on what was done (one line is fine)
- Stop and wait before moving to the next task unless instructed otherwise

Do not refactor, rename, or restructure anything not mentioned in the task. Read the relevant existing files before writing any code.

---

## Constraints

- Repo: https://github.com/adilio/PromptStash
- Deployed: https://promptstash.netlify.app
- Do not change the Supabase client setup in `src/lib/supabase.ts`
- Do not rename existing components unless a task explicitly says to
- Do not change the Tailwind config or shadcn theme tokens unless a task explicitly says to
- All changes must pass `npm run lint` and `npm run build` before being marked complete

## Git instructions

After each completed task, commit and push directly to `main`. No branches, no PRs.

- Commit as the repo owner identity — do not add any Claude, co-author, or AI attribution to commits
- Commit message format: short imperative sentence describing what changed, e.g. `Swap sanitize-html for DOMPurify`
- Do not include `Co-authored-by`, `Generated by`, or any similar trailer lines in commit messages
- Push immediately after committing — do not batch multiple tasks into one commit
